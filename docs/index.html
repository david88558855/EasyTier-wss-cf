Core capabilities

This is not a proxy. It is a direct Cloudflare Workers implementation for EasyTier WSS.

Apple-style · clean · lightweight

Management panel

Create, edit, and delete routes in `/panel`, with copyable EasyTier commands.

Route coordination

`RouteHub` Durable Object handles connection coordination within each route.

Simple deployment

Fork on GitHub, then connect the repo in Cloudflare Workers web UI. No local deployment is required.

Architecture

The project is composed of a Worker, a storage DO, and a route hub DO, each with a clear responsibility.

Control plane

Storage and login

`AdminStore` stores routes, stats, and events. The panel password is configured as a Cloudflare Workers Secret, and `ADMIN_PASSWORD` is recommended.

POST /api/login { "password": "your ADMIN_PASSWORD" } GET /api/state Returns routes, stats, and events
Connection plane

WSS entry and peer coordination

Every route uses `/ws/:routeId/:clientToken` as the public entry. Workers parse EasyTier peer headers and coordinate messages between connections in the same route.

wss://your-domain/ws/route-id/client-token RouteHub manages the route connection set

Deployment

Recommended flow: fork the repo on GitHub, then deploy it from the Cloudflare Workers web UI. No local commands are required.

Fork

Copy the repo to your GitHub account

Open the repository page and click Fork. Cloudflare will connect directly to your forked repo later.

Cloudflare

Import the GitHub repo in Workers

Open the Cloudflare Workers creation page, choose GitHub import, then authorize and deploy.

Secret

Set the panel password

Add `ADMIN_PASSWORD` in Secrets / Variables. That becomes the management panel password.

Done

Open `/panel` and create routes

After deployment, open `/panel`, sign in, and create routes to generate EasyTier WSS commands.

Panel and usage

The panel manages routes and generates client commands, which helps reduce configuration mistakes.

What the panel does
  • ✓Route management：Create, edit, delete, enable, or disable routes.
  • ✓Command generation：Generate an EasyTier client command you can copy directly.
  • ✓Status view：Inspect connection count, totals, and recent events.
Example command
sudo easytier-core -d \ --network-name my-network \ --network-secret my-secret \ -p 'wss://your-domain/ws/route-id/client-token' Copy the generated command from the panel

FAQ

A quick place to confirm the most common deployment and positioning questions.

Is this a proxy for another WSS service?

No. It is a direct Cloudflare Workers implementation for EasyTier WSS.

How do I set the panel password?

Use Cloudflare Workers Secrets / Variables to set ADMIN_PASSWORD. The code also accepts ADMIN_SECRET for compatibility.

Why recommend GitHub Fork + Cloudflare web deployment?

Because this keeps the source in GitHub and lets Cloudflare deploy directly from the repository, with no local build or deploy step required.

How do I enable this GitHub Pages site?

In repository Settings → Pages, choose Deploy from a branch, set branch to main, and directory to /docs. GitHub Pages will serve this site automatically.
